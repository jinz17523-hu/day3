# C语言转义字符速查表

> 学习日期：2026-07-08  
> 来源：《C程序设计语言》及个人实践

## 常用转义序列

| 转义序列 | 含义                     | 说明                                                                 |
|----------|--------------------------|----------------------------------------------------------------------|
| `\?`     | 防止被解析为三字母词      | 在连续书写 `??` 时使用，新编译器已不再支持三字母词，故无法验证效果。 |
| `\'`     | 单引号字符常量            | 用于表示字符常量中的单引号，例如 `'\''` 表示字符 `'`。               |
| `\"`     | 双引号                    | 用于表示字符串内部的英文双引号，例如 `"He said \"Hi\""`。            |
| `\\`     | 反斜杠字符                | 用于表示一个真正的反斜杠，防止被解释为转义序列的开始。               |
| `\a`     | 警报（响铃/闪烁）         | 终端发出警报声或闪烁，现代终端可能同时触发两者。                     |
| `\b`     | 退格                      | 光标回退一个字符，但不删除该字符（只是移动光标位置）。               |
| `\f`     | 换页                      | 光标移到下一页，现代系统多表现为类似 `\v` 的行为（垂直移动）。       |
| `\n`     | 换行                      | 光标移动到下一行的开头（最常用的换行符）。                           |
| `\r`     | 回车                      | 光标回到当前行的开头，不换行。                                       |
| `\t`     | 水平制表符                | 光标跳到下一个水平制表位，通常每 4 或 8 个字符一个制表位。           |
| `\v`     | 垂直制表符                | 光标向下移动到下一行的同一列（垂直方向上的制表）。                   |

## 注意事项

- **三字母词**：历史上某些老式编译器会将 `??` 开头的序列解释为其他字符（如 `??=` 变成 `#`），因此引入 `\?` 来阻止这种替换。现代编译器（如 GCC、MSVC 2010+）默认已禁用三字母词，所以 `\?` 实际上等同于普通 `?`。
- **`\a` 效果**：取决于终端模拟器，大多数现代终端会发出系统提示音（beep），部分图形终端会闪烁窗口。
- **`\f` 与 `\v`**：在纯文本环境下很少用到，更多见于打印机或早期终端协议；现代控制台中 `\f` 行为可能与 `\v` 相似（垂直移动一行）。

## 示例代码片段
include <stdio.h>

int main() {

// 打印数字123，制表符，长数字串，制表符，数字5

printf("123\t12123456345\t5");

printf("%c\n",'x');

return 0;

}
